# 이미지 가이드라인

LECPIN 디자인 시스템의 이미지 사용 원칙입니다.
AI 티를 배제하고, 사진 중심의 진정성 있는 비주얼을 지향합니다.

---

## 핵심 원칙

### 1. 실제 사진 우선
- 3D 렌더링, 일러스트, 아이콘 그래픽 대신 **실제 사진** 사용
- 스톡 사진보다 **실제 학습 환경, 실제 사람**이 나온 사진 선호
- 연출된 느낌보다 **자연스러운 순간 포착**

### 2. 그라데이션/이모지 금지
- 배경에 그라데이션 절대 사용 금지
- 이모지, 아이콘 최소화 (필요시 텍스트로 대체)
- "AI스러운" 보라색/파란색 그라데이션 완전 배제

### 3. 에디토리얼 톤
- 매거진/잡지 사진 같은 퀄리티
- 적절한 여백과 함께 사진 배치
- 사진 위에 텍스트 오버레이 최소화

## 이미지 유형별 가이드

### Hero 이미지 (16:9)
```
용도: 메인 페이지 상단, 랜딩 페이지
```

**추천 사진 예시:**
- 노트북 앞에서 집중해서 코딩하는 사람 (얼굴보다 손/화면 위주)
- 카페나 도서관에서 학습하는 자연스러운 모습
- 깔끔한 데스크 셋업 (미니멀한 환경)
- 화이트보드에 다이어그램 그리며 토론하는 장면

**피해야 할 것:**
- 과도하게 연출된 포즈
- 뻔한 "엄지 척" 스톡 사진
- 화려한 네온/RGB 조명

---

### 카드 썸네일 (4:3)
```
용도: 강의 카드, 콘텐츠 미리보기
```

**추천 사진 예시:**
- 코드가 보이는 모니터 클로즈업 (블러 처리된 코드)
- 노트에 필기하는 손
- 키보드 타이핑하는 손 클로즈업
- 간단한 다이어그램이나 스케치

**피해야 할 것:**
- 복잡한 구도
- 여러 사람이 동시에 나오는 사진
- 텍스트가 많이 보이는 화면

---

### 프로필/아바타 (1:1)
```
용도: 사용자 프로필, 멘토 소개
```

**추천 사진 예시:**
- 자연광 아래 자연스러운 표정
- 중립적인 배경 (흰색, 회색, 자연 배경)
- 어깨 위로 크롭된 구도

**피해야 할 것:**
- 과도한 필터/보정
- 복잡한 배경
- 정면 응시하며 웃는 스톡 사진 느낌

---

### 피처 이미지 (21:9 와이드)
```
용도: 섹션 구분, 배너
```

**추천 사진 예시:**
- 넓은 코워킹 스페이스 풍경
- 책상 위 아이템들 평면 배치 (flat lay)
- 창밖 풍경이 보이는 작업 환경
- 미니멀한 오브젝트 (커피잔, 노트, 펜)

## 색상 처리

### 기본 원칙
- 원본 색감 유지 (과도한 보정 금지)
- 채도는 자연스럽게, 약간 낮추는 것도 OK

### 허용되는 처리
```css
/* 미세한 필름 느낌 */
.img-editorial {
  filter: contrast(1.02) saturate(0.95);
}

/* 흑백 처리 (특별한 경우) */
.img-bw {
  filter: grayscale(1) contrast(1.1);
}
```

### 금지되는 처리
- 컬러 오버레이 (특히 보라색, 파란색)
- 그라데이션 오버레이
- 과도한 비네팅
- HDR 효과

## 플레이스홀더 사용법

실제 이미지가 준비되기 전까지 플레이스홀더를 사용합니다.

```jsx
{/* Hero 이미지 플레이스홀더 */}
<div className="img-placeholder img-hero">
  {/*
    권장 이미지: 노트북 앞에서 코딩하는 사람의 옆모습
    - 자연광이 들어오는 창가 환경
    - 화면에는 코드가 살짝 보이되 읽히지 않을 정도로
    - 집중하는 자연스러운 표정
    출처 추천: Unsplash "developer working", "coding natural light"
  */}
</div>

{/* 카드 썸네일 플레이스홀더 */}
<div className="img-placeholder img-card">
  {/*
    권장 이미지: 모니터에 보이는 코드 클로즈업
    - 구문 하이라이팅이 보이는 에디터 화면
    - 살짝 흐릿하게 (depth of field)
    - 다크 테마 에디터 선호
    출처 추천: Unsplash "code editor screen"
  */}
</div>

{/* 프로필 플레이스홀더 */}
<div className="img-placeholder img-square">
  {/*
    권장 이미지: 자연스러운 프로필 사진
    - 중립적인 배경
    - 자연광
    - 편안한 표정
  */}
</div>
```

## 추천 이미지 소스

### 무료 스톡
1. **Unsplash** - 고퀄리티 무료 사진
   - 검색어: "developer", "coding", "workspace minimal"
2. **Pexels** - 다양한 직업/학습 장면
3. **Pixabay** - 상업용 무료

### 유료 스톡 (더 나은 퀄리티)
1. **Getty Images** - 프리미엄 품질
2. **Shutterstock** - 다양한 선택지
3. **Adobe Stock** - 고퀄리티

### 직접 촬영 권장
- 실제 서비스 사용자/학습자 촬영
- 실제 학습 환경 촬영
- 진정성 있는 비주얼 확보

## 접근성 고려사항

### Alt 텍스트 필수
```jsx
<img
  src="/images/hero.jpg"
  alt="밝은 창가에서 노트북으로 코딩하며 학습하는 사람"
  className="img-cover img-editorial"
/>
```

### 장식용 이미지
```jsx
{/* 순수 장식용 이미지는 빈 alt */}
<img src="/images/decoration.jpg" alt="" role="presentation" />
```

### 색상 대비
- 이미지 위 텍스트는 최소화
- 불가피할 경우 반투명 배경 처리

## 파일 형식 및 최적화

### 권장 형식
- **WebP**: 기본 포맷 (용량 대비 품질 최고)
- **AVIF**: 최신 브라우저용 (더 나은 압축)
- **JPG**: 폴백용

### 해상도
- Hero: 1920px 너비
- Card: 800px 너비
- Avatar: 400px 너비 (200px @2x)

### Next.js Image 컴포넌트 사용
```jsx
import Image from 'next/image'

<Image
  src="/images/hero.webp"
  alt="코딩하는 사람"
  width={1920}
  height={1080}
  className="img-cover img-editorial"
  priority // Hero 이미지는 priority 추가
/>
```
